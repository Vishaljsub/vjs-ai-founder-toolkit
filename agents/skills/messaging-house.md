# Skill: Messaging House

**Run by:** positioning-strategist → content-engine · **Output to:** `../workspace/01-positioning/messaging-house.md`

A one-page source of truth every asset pulls from. Lock it before writing copy.

```
                 ┌────────────────────────────────────────┐
   ROOFLINE →    │  Tagline / one-liner everyone repeats     │
                 │  "{{ONE_LINE_TAGLINE}}"                    │
                 ├─────────────┬─────────────┬──────────┤
   PILLARS →     │  Pillar 1     │  Pillar 2     │  Pillar 3 │
                 │  {{PILLAR_1}} │  {{PILLAR_2}} │ {{PILLAR_3}} │
   PROOF →       │  proof 1      │  proof 2      │  proof 3  │
                 └─────────────┴─────────────┴──────────┘
   FOUNDATION →  Proof points, demo moments, [customer proof — needs proof]
```

## Fill these in
- **One-liner:** the sentence a champion repeats to their boss.
- **3 pillars:** each = a buyer benefit, not a feature category.
- **Proof under each pillar:** a real capability or demo moment. Mark customer proof `[needs proof]`.
- **Elevator pitch (30 sec):** problem → {{PRODUCT_NAME}} → why different → ask.
- **Boilerplate (50 words):** reusable for site footer, decks, PR.

### Done when
- Fits on one page, every claim traceable, and content-engine can write any asset from it without inventing.
